
# じゃんけんカードVR

ハンドトラッキングを使いでカードを**物理的に掴んで**モンスターを召喚・バトルさせるVRカードゲームです。

## ゲームシステム設計

### フェーズステートマシン

1ターンは以下の5フェーズで構成され、`PhaseController` が全体を統括します。

```
startPhase（手札5枚ドロー）
    ↓
summonPhase（ベンチにカードを3枚召喚）
    ↓
selectPhase（バトルに出す1枚を選択）
    ↓
battlePhase（じゃんけん判定・バトルアニメーション）
    ↓
drawPhase（負け or 引き分け時に1枚補充）
    ↓ ループ or ゲーム終了
```

### VRインタラクション設計

```
プレイヤーがカードをグラブ
    ↓
CardColliderEvent（ベンチゾーン）が OnTriggerStay で検出
    ↓
IsGrabOn で「グラブを離したか」を判定
    ↓ 離した瞬間
カードをスナップ固定 → 3Dモンスターモデルを生成（召喚演出）
```

- VRコントローラーで「物理的に掴んで置く」操作感を実現するため  
  **レイキャスト方式ではなく物理コライダー方式**を採用。
- グラブ中のカード誤検出を防ぐため `IsGrabOn` コンポーネントで  
  「グラブを離した瞬間」のみスナップ処理が走る設計にしました。

### カードデータ管理

```
テクスチャ名（文字列）
    → Dictionary で O(1) ルックアップ
    → 3Dモデル / じゃんけんの手 / SE / アニメーション数 を取得
```

カードのプレハブは1種類（`OriginalCardPrefab`）だけ用意し、  
マテリアルの `_MainTex2` スロットにテクスチャを動的に差し替えることで  
複数種類のカードを管理しています。

## 工夫した点・苦労した点

### ハンドトラッキングによるカード操作

Meta Quest 2 のハンドトラッキングで「カードを引く」「ベンチに置く」という  
自然な手の動作を実現することが最大の挑戦でした。

- **課題**：手でカードを掴んだまま誤検出されてカードが固定されてしまう
- **解決**：`IsGrabOn` で「グラブ中か否か」のフラグを管理し、  
  `OnTriggerStay` 内で「グラブを離した瞬間のみ」スナップが発動する仕組みに設計

### フェーズ管理の複雑さへの対処

VRならではの「プレイヤーがいつカードを動かすか予測できない」問題に対して、  
Coroutine の `while (phase) { yield return null; }` ループで  
フラグが変わるまで待機し続けることで、入力タイミングに依存しない  
堅牢なゲームフロー制御を実現しています。
